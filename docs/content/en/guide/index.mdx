# Introduction to Byonk

**Byonk** (Bring Your Own Ink) is a self-hosted content server for [TRMNL](https://usetrmnl.com) e-ink devices. It lets you create custom screens using Lua scripts and SVG templates, giving you complete control over what your e-ink display shows.

![Example screen showing transit departures](/samples/transit.png)

## Why Byonk?

TRMNL devices are beautiful e-ink displays, but the default cloud service may not fit everyone's needs:

- **Self-hosted**: Keep your data on your own server
- **Customizable**: Create any screen you can imagine
- **Flexible**: Fetch data from any API or website
- **Efficient**: Smart refresh rates optimize battery life

## Features

### Lua Scripting

Write simple Lua scripts to fetch and process data:

```lua
local response = http_get("https://api.example.com/data")
local data = json_decode(response)
return {
  data = { items = data.results },
  refresh_rate = 300
}
```

### SVG Templates

Design screens using familiar SVG with Tera templating:

```svg
<svg viewBox="0 0 800 480">
  <text x="20" y="40">{{ title }}</text>
  {% for item in items %}
    <text y="{{ 80 + loop.index0 * 30 }}">{{ item.name }}</text>
  {% endfor %}
</svg>
```

### E-ink Optimized

- 4-level grayscale with Floyd-Steinberg dithering
- Variable font support for crisp text
- Automatic sizing for different TRMNL models

### Device Management

Map different screens to different devices via YAML configuration:

```yaml
devices:
  "94:A9:90:8C:6D:18":
    screen: transit
    params:
      station: "Olten, Bahnhof"
```

## Requirements

- **Rust** (for building from source)
- A **TRMNL device** configured to point to your server
- Network connectivity between device and server
