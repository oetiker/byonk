# Byonk ↔ HA Phase 4 — Release, Validation & Docs Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Validate the byonk add-on + integration end-to-end on a scripted headless Home Assistant OS VM, then automate add-on/integration versioning, prepare HACS/brands distribution, and align docs.

**Architecture:** A committed `tools/ha-vm/` harness boots HAOS `generic-aarch64` headless under QEMU (hvf-accelerated) with host-forwarded ports and a Samba-based deploy loop. A tested `tools/release/bump-addon-version.sh` is wired into a post-image-publish release job. A generated byonk icon + CI validation make the integration HACS/brands-ready; external PRs are drafted for the maintainer to file. Docs are updated to the shipped state.

**Tech Stack:** Bash, QEMU (`qemu-system-aarch64`), Home Assistant OS, Samba, GitHub Actions, `rsvg-convert`/`sips`, mdBook.

## Global Constraints

- Host is **macOS arm64 (Apple M1 Max)**; QEMU resolves from `$(brew --prefix qemu)/bin` or the UTM bundle; EFI code from `$(brew --prefix qemu)/share/qemu/edk2-aarch64-code.fd` or `/Applications/UTM.app/Contents/Resources/qemu/edk2-aarch64-code.fd`.
- The integration is **Supervised/HAOS-only** — the test bed must be HAOS (not HA Container).
- **Zero-touch / no-redundancy trust**: the admin token's only home is the add-on option; the config entry stores no token.
- Byonk `config.yaml` is the **source of truth** the integration mirrors.
- **No byonk Rust runtime changes** in this phase.
- The add-on `version:` **must equal an already-published `ghcr.io/oetiker/byonk` tag** → bump it only *after* the image is pushed.
- Shell scripts: `set -euo pipefail`, portable across BSD (macOS, dev) and GNU (CI) — prefer `awk` over GNU-only `sed` constructs; `sed -i.bak … && rm -f *.bak` is portable.
- Do **not** `git add -A`/`.` — add explicit paths; large VM artifacts are git-ignored.

- [ ] **Step 1: Write `tools/ha-vm/lib.sh`**

```bash
#!/usr/bin/env bash
# Shared paths + tool resolution for the byonk HAOS test harness.
set -euo pipefail

HA_VM_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
REPO_ROOT="$(cd "$HA_VM_DIR/../.." && pwd)"
WORK_DIR="${BYONK_HA_VM_WORK:-$HA_VM_DIR/work}"
DISK="$WORK_DIR/haos.qcow2"
VARS="$WORK_DIR/vars.fd"
HAOS_VERSION_FALLBACK="15.2"

haos_version() {
  if [ -n "${HAOS_VERSION:-}" ]; then echo "$HAOS_VERSION"; return; fi
  local tag
  tag="$(curl -fsSL https://api.github.com/repos/home-assistant/operating-system/releases/latest \
        | sed -n 's/.*"tag_name": *"\([^"]*\)".*/\1/p' | head -1 || true)"
  echo "${tag:-$HAOS_VERSION_FALLBACK}"
}

resolve_tools() {
  local qprefix qbin efi
  qbin=""
  if command -v qemu-system-aarch64 >/dev/null 2>&1; then
    qbin="$(command -v qemu-system-aarch64)"
  elif qprefix="$(brew --prefix qemu 2>/dev/null)" && [ -x "$qprefix/bin/qemu-system-aarch64" ]; then
    qbin="$qprefix/bin/qemu-system-aarch64"
  elif [ -x /Applications/UTM.app/Contents/Resources/qemu/qemu-system-aarch64 ]; then
    qbin="/Applications/UTM.app/Contents/Resources/qemu/qemu-system-aarch64"
  fi
  [ -n "$qbin" ] || { echo "ERROR: qemu-system-aarch64 not found (try: brew install qemu)" >&2; exit 1; }
  QEMU="$qbin"

  efi=""
  for cand in \
    "$(brew --prefix qemu 2>/dev/null)/share/qemu/edk2-aarch64-code.fd" \
    "/Applications/UTM.app/Contents/Resources/qemu/edk2-aarch64-code.fd"; do
    [ -f "$cand" ] && { efi="$cand"; break; }
  done
  [ -n "$efi" ] || { echo "ERROR: edk2-aarch64-code.fd firmware not found" >&2; exit 1; }
  EFI_CODE="$efi"
  export QEMU EFI_CODE
}
```

- [ ] **Step 2: Write `tools/ha-vm/fetch.sh`**

```bash

# Download + decompress the HAOS generic-aarch64 image into the work dir.

mkdir -p "$WORK_DIR"

if [ -f "$DISK" ]; then
  echo "Disk already present: $DISK (delete it or run 'make ha-vm-clean' to re-fetch)"
  exit 0
fi

VER="$(haos_version)"
ASSET="haos_generic-aarch64-${VER}.qcow2.xz"
URL="https://github.com/home-assistant/operating-system/releases/download/${VER}/${ASSET}"
echo "Fetching HAOS ${VER} from ${URL}"
curl -fL --retry 3 -o "$WORK_DIR/$ASSET" "$URL"
echo "Decompressing…"
xz -dc "$WORK_DIR/$ASSET" > "$DISK"
rm -f "$WORK_DIR/$ASSET"
echo "Ready: $DISK"
```

- [ ] **Step 1: Write `tools/ha-vm/run.sh`**

```bash
#!/usr/bin/env bash
# Boot the byonk HAOS test VM headless. Ctrl-A X (then Enter) to quit the serial console.
set -euo pipefail
source "$(dirname "${BASH_SOURCE[0]}")/lib.sh"
resolve_tools

[ -f "$DISK" ] || bash "$HA_VM_DIR/fetch.sh"

# Prepare a writable UEFI varstore matching the firmware size.
if [ ! -f "$VARS" ]; then
  vtmpl="$(brew --prefix qemu 2>/dev/null)/share/qemu/edk2-arm-vars.fd"
  if [ -f "$vtmpl" ]; then
    cp "$vtmpl" "$VARS"
  else
    truncate -s "$(stat -f%z "$EFI_CODE")" "$VARS"
  fi
fi

echo "HA UI:  http://localhost:8123   (first boot takes several minutes)"
echo "byonk:  http://localhost:3000"
exec "$QEMU" \
  -name byonk-haos \
  -M virt,accel=hvf,highmem=on \
  -cpu host -smp "${VM_CPUS:-4}" -m "${VM_RAM_MB:-4096}" \
  -drive "if=pflash,format=raw,readonly=on,file=$EFI_CODE" \
  -drive "if=pflash,format=raw,file=$VARS" \
  -drive "if=virtio,format=qcow2,file=$DISK" \
  -netdev user,id=net0,hostfwd=tcp::8123-:8123,hostfwd=tcp::3000-:3000,hostfwd=tcp::4445-:445 \
  -device virtio-net-pci,netdev=net0 \
  -device virtio-rng-pci \
  -display none -serial mon:stdio
```

```bash
#!/usr/bin/env bash
# rsync the byonk integration into the running VM's /config via the Samba add-on.
set -euo pipefail
source "$(dirname "${BASH_SOURCE[0]}")/lib.sh"

: "${SMB_USER:?set SMB_USER to the Samba add-on username}"
: "${SMB_PASS:?set SMB_PASS to the Samba add-on password}"

mnt="$(mktemp -d)"
cleanup() { umount "$mnt" 2>/dev/null || true; rmdir "$mnt" 2>/dev/null || true; }
trap cleanup EXIT

mount -t smbfs "//${SMB_USER}:${SMB_PASS}@localhost:4445/config" "$mnt"
mkdir -p "$mnt/custom_components/byonk"
rsync -a --delete \
  --exclude '__pycache__' --exclude '*.pyc' \
  "$REPO_ROOT/custom_components/byonk/" "$mnt/custom_components/byonk/"
echo "Deployed. Restart HA (Developer Tools → Restart, or 'ha core restart' on the serial console)."
```

- [ ] **Step 3: Write `tools/release/bump-addon-version.sh`**

```bash
#!/usr/bin/env bash
# Bump the HA add-on version + changelog. Portable across BSD/GNU.
set -euo pipefail
VERSION="${1:?usage: bump-addon-version.sh <version> [--breaking]}"
MODE="${2:-}"
ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/../.." && pwd)"
CFG="${ADDON_CONFIG:-$ROOT/homeassistant/byonk/config.yaml}"
CHANGELOG="${ADDON_CHANGELOG:-$ROOT/homeassistant/byonk/CHANGELOG.md}"

sed -i.bak -E "s/^version: .*/version: \"${VERSION}\"/" "$CFG" && rm -f "$CFG.bak"

if [ "$MODE" = "--breaking" ]; then
  if grep -q '^breaking_versions:' "$CFG"; then
    awk -v v="$VERSION" '/^breaking_versions:/{print; print "  - \"" v "\""; next} {print}' \
      "$CFG" > "$CFG.tmp" && mv "$CFG.tmp" "$CFG"
  else
    printf 'breaking_versions:\n  - "%s"\n' "$VERSION" >> "$CFG"
  fi
fi

awk -v ver="$VERSION" '
  NR==1 && /^# Changelog/ { print; print ""; print "## " ver; print ""; print "- Updated to byonk " ver "."; next }
  { print }
' "$CHANGELOG" > "$CHANGELOG.tmp" && mv "$CHANGELOG.tmp" "$CHANGELOG"
```
